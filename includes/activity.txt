Meme-tastic data science activity: OpenAI API + Tidyverse for graduate students

Overview
- Goal: Learn to call OpenAI’s API from R, handle and parse responses, and analyze model-generated captions using tidyverse workflows.
- Theme: Create witty meme captions for a small meme dataset, then analyze how prompts and settings influence outputs.
- Deliverables: A reproducible R script/notebook, a tidy-caption table, and a set of visual analytics comparing captions across memes.

Learning objectives
- Build a small data pipeline in R that calls the OpenAI API and handles JSON responses.
- Engineer prompts and experiment with API parameters (temperature, max_tokens) to shape outputs.
- Use tidyverse (dplyr, purrr, ggplot2, tidyr) to generate, tidy, and summarize caption results.
- Perform basic text-analysis on captions (word counts, basic sentiment) using tidytext or equivalent.
- Interpret how prompt design and model settings affect humor, length, and relevance of captions.
- Document a reproducible workflow and reflect on limitations, biases, and cost considerations.

Prerequisites
- R and RStudio (or any R environment)
- Internet access
- OpenAI API key (set as an environment variable OPENAI_API_KEY)
- R packages: tidyverse, httr, jsonlite, purrr, tidytext (optional but recommended)

Materials
- A short meme dataset (descriptions of memes you’ll caption)
- An OpenAI API key
- An optional rubric for humor/relevance (for evaluation)

Activity outline (5–6 steps)

1) Set up and data
- Install and load packages
- Create a small meme dataset with descriptions (no images needed for caption generation)
- Example meme dataset (6 memes)

2) Build an API call helper
- Write a helper function to call OpenAI’s chat/completions endpoint, extract the caption, and capture token usage for cost awareness
- Use a system role prompt to steer the style (concise, witty, <= 15 words)

3) Generate captions
- For each meme, create a prompt that asks for a single witty caption
- Call the API for each meme (respect rate limits; consider Sys.sleep if needed)
- Collect results into a tidy table: meme_id, meme_name, description, caption, tokens_used

4) Analyze captions with tidyverse
- Compute caption length (word count)
- Basic text processing: split captions into words, count tokens, etc.
- Optional sentiment analysis using tidytext (e.g., Bing lexicon) to get a rough sentiment score per caption
- Create visualizations:
  - Caption length by meme
  - Distribution of sentiment across captions
  - Wordcloud or top words (optional)

5) Prompt engineering and comparisons (optional)
- Repeat Step 3 with alternative system prompts or temperature settings
- Compare how outputs change (length, humor, and relevance)
- Summarize findings: which prompt settings produced the most on-target captions?

6) Reflection and deliverables
- Discussion prompts: How does temperature affect creativity? How consistent are captions with meme content? What errors or biases appear?
- Deliverables: the R script/notebook, the caption table, and a short interpretation write-up.

# Install and load packages
# install.packages(c("tidyverse","httr","jsonlite","purrr","tidytext"))
library(tidyverse)
library(httr)
library(jsonlite)
library(purrr)
library(tidytext)

# Set API key (alternative: Sys.setenv(OPENAI_API_KEY = "YOUR_KEY_HERE"))
Sys.setenv(OPENAI_API_KEY = "YOUR_KEY_HERE")

# 1) Meme dataset (6 memes)
memes <- tibble(
  id = 1:6,
  meme_name = c("Distracted Boyfriend",
                "Drake Hotline Bling",
                "Two Buttons",
                "Expanding Brain",
                "Doge",
                "Mocking SpongeBob"),
  description = c(
    "A man looks at another woman while his girlfriend watches in disbelief.",
    "Drake rejects something in the top image and approves something in the bottom image.",
    "A person faces two red buttons and is unsure which to press.",
    "Brain icon expands through levels of thought from basic to cosmic.",
    "Shiba Inu with multicolored, broken-English captions about internet culture.",
    "Spongebob imitates a mocking tone in alternating uppercase/lowercase."
  )
)

# 2) OpenAI caption helper
openai_caption <- function(prompt, model = "gpt-3.5-turbo", temp = 0.7, max_tokens = 60){
  url <- "https://api.openai.com/v1/chat/completions"
  body <- list(
    model = model,
    messages = list(
      list(role = "system", content = "You are a witty meme-caption generator. Write concise captions in no more than 15 words."),
      list(role = "user", content = prompt)
    ),
    temperature = temp,
    max_tokens = max_tokens
  )
  res <- POST(url,
              add_headers(Authorization = paste("Bearer", Sys.getenv("OPENAI_API_KEY"))),
              body = toJSON(body, auto_unbox = TRUE),
              encode = "json")
  stop_for_status(res)
  out <- content(res, as = "text", encoding = "UTF-8") %>% fromJSON(flatten = TRUE)
  caption <- out$choices[[1]]$message$content
  tibble(
    raw = list(out),
    caption = caption,
    tokens_used = if (!is.null(out$usage$total_tokens)) out$usage$total_tokens else NA_integer_
  )
}

# 3) Prepare prompts for each meme
memes <- memes %>% mutate(
  prompt = map2_chr(meme_name, description,
                    ~ str_glue("For the meme '{.x}' described as: \"{.y}\". Provide a single witty caption (<=15 words).")
  )
)

# 4) Generate captions (respect API rate limits)
# Note: you can vary temperature or max_tokens to experiment
captions <- memes %>% 
  mutate(result = map2(prompt, id, ~ openai_caption(.x, max_tokens = 60, temp = 0.7))) %>% 
  mutate(caption = map_chr(result, "caption"),
         tokens_used = map_int(result, "tokens_used"))

# 5) Analyze captions
captions_df <- captions %>% select(id, meme_name, description, caption, tokens_used)

# caption length (in words)
caption_words <- captions_df %>% 
  mutate(words = map_int(str_split(caption, "\\s+"), length))

# Optional: simple sentiment (using tidytext and Bing lexicon)
sentiment_df <- caption_words %>% 
  unnest_tokens(word, caption) %>%
  inner_join(get_sentiments("bing"), by = "word") %>%
  group_by(meme_name) %>%
  summarize(sentiment = sum(ifelse(sentiment == "positive", 1, -1))) %>% 
  right_join(caption_words %>% select(meme_name) %>% distinct(), by = "meme_name") %>%
  replace_na(list(sentiment = 0))

analysis <- caption_words %>% 
  left_join(sentiment_df, by = "meme_name")

# 6) Visualizations
# a) Caption length by meme
ggplot(caption_words, aes(x = meme_name, y = words)) +
  geom_col() +
  coord_flip() +
  labs(title = "Caption length by meme", x = "Meme", y = "Word count")

# b) Sentiment by meme (if computed)
ggplot(analysis, aes(x = meme_name, y = sentiment)) +
  geom_col(fill = "steelblue") +
  coord_flip() +
  labs(title = "Caption sentiment by meme (approx.)", x = "Meme", y = "Sentiment score")

Notes and best practices
- API usage and cost: track tokens_used per caption to estimate costs.
- Rate limits: insert Sys.sleep(0.5–1) between requests if needed.
- Error handling: wrap API calls in tryCatch to gracefully handle network errors or rate limits.
- Reproducibility: store prompts and results in a project folder; consider saving a results.rds for later analysis.
- Ethics and safety: avoid sharing sensitive data; comply with OpenAI usage policies; critically assess outputs for bias and humor appropriateness.

Extensions (optional)
- Compare endpoints: experiment with completions vs. chat (gpt-4o if available) and adjust prompts accordingly.
- Improve evaluation: add human-labeled humor scores from a rubric or use multiple prompts to generate several caption options per meme, then crowdsource a best caption.
- Deeper text analysis: use tidytext to compute lexical diversity, n-grams, or apply a more robust sentiment model (e.g., syuzhet, NRC) if desired.
- Visualization improvements: facet by meme type, or create a small dashboard in RShiny to interactively explore caption metrics.
