Meme-tastic Data Science with OpenAI API in Tidyverse
Overview
Graduate students will build a small, end-to-end data workflow: craft meme-worthy prompts about data science topics, call OpenAI’s API from R (tidyverse-friendly), handle and parse the JSON responses, and analyze the captions with tidytext and ggplot2. The aim is to practice API calls, response handling, text analysis, and visualization in a reproducible, tidyverse-centric workflow—with a dash of memes.

Learning objectives
- Set up and authenticate to OpenAI’s API from R.
- Compose structured prompts (including system and user roles) and make chat completions.
- Extract and clean API responses, then store results in a tidy data frame.
- Perform basic text analytics on generated captions (tokenization, word counts, sentiment).
- Compare prompts and visualize differences in outputs.
- Reflect on prompt engineering, model behavior, and reproducibility considerations.

Prerequisites
- R and RStudio (or any R environment)
- Packages: tidyverse, openai, tidytext, textdata (optional for sentiment lexicons), stringr
- OpenAI API key (set as environment variable OPENAI_API_KEY)
- Basic familiarity with the tidyverse (dplyr, purrr, tidyr, ggplot2, tibble)

Setup and authentication (one-time)
- Install packages (if needed):
  - install.packages(c("tidyverse", "openai", "tidytext", "textdata"))
- Load libraries and set API key:
  - library(tidyverse)
  - library(openai)
  - library(tidytext)
  - Sys.setenv(OPENAI_API_KEY = Sys.getenv("OPENAI_API_KEY"))  # ensure your key is set

- Optional: set a sane max_tokens to avoid runaway costs:
  - options(openai.max_tokens = 80)

Activity outline (4–6 person teams, ~2–3 hours)
1) Prepare the topic prompts (instructor provides a short list or students propose)
- We'll use a small set of data-science topics to generate memes about.
- Each topic will have an associated tone (nerdy, dry wit, meme-friendly, etc.)

2) Define a reusable API call function (OpenAI chat completions)

3) Generate captions via API calls

4) Analyze the results
- Compute caption length, tokenize and find top words, and estimate sentiment.
- Compare captions across topics and/or across different prompt styles (if you run multiple prompts).

5) Deliverables
- A tidy dataframe with topic, caption, length, and sentiment.
- A simple report or slide deck with plots and interpretation.
- A short memo reflecting on the prompt design decisions and observed model behavior.

2) Define a function to generate a caption via OpenAI
- We’ll use the OpenAI R package’s chat completion. The function builds a prompt and calls the API, returning the caption text.

```R
# Ensure you have: library(openai)
library(openai)

# Simple helper to trim whitespace
trim <- function(x) gsub("^\\s+|\\s+$", "", x)

generate_caption <- function(topic, tone = "nerdy",
                             model = "gpt-4",
                             max_tokens = 60,
                             temperature = 0.8) {

  prompt <- paste0(
    "Create a single, meme-ready data science caption about '", topic,
    "'. The caption should be witty, concise, and tailored to data scientists. ",
    "Use data-science jargon where appropriate and keep it under about 60 words. ",
    "Tone: ", tone,
    ". Output only the caption text, no extra commentary."
  )

  resp <- create_chat_completion(
    model = model,
    messages = list(
      list(role = "system",
           content = "You are a meme-capable data scientist assistant who crafts witty captions for memes about data science topics."),
      list(role = "user", content = prompt)
    ),
    max_tokens = max_tokens,
    temperature = temperature
  )

  caption <- resp$choices[[1]]$message$content
  trim(caption)
}
```

generate_caption_variant <- function(topic, variant = 1, model = "gpt-4", max_tokens = 60, temperature = 0.8) {
  if (variant == 1) {
    prompt <- paste0("Create a short, punchy meme caption about '", topic,
                     "'. Keep it under 8 words if possible, but humorous and clear for data scientists.")
  } else {
    prompt <- paste0("Write a witty data-science caption about '", topic,
                     "'. Use a bit more sarcasm and nerdy jargon.")
  }

  resp <- create_chat_completion(
    model = model,
    messages = list(
      list(role = "system",
           content = "You are a meme-capable data scientist assistant who crafts witty captions for memes about data science topics."),
      list(role = "user", content = prompt)
    ),
    max_tokens = max_tokens,
    temperature = temperature
  )

  trim(resp$choices[[1]]$message$content)
}

5) Text analysis with tidytext (tokenization and sentiment)
- We’ll compute basic metrics: word frequencies, and a simple sentiment score for each caption.

```R
library(tidytext)

# Use a single caption per topic for simplicity; you can expand to multiple captions
captions_for_analysis <- captions %>% select(id, topic, caption)

# Tokenize and compute sentiment using Bing lexicon
sentiment <- captions_for_analysis %>%
  unnest_tokens(word, caption) %>%
  anti_join(stop_words, by = "word") %>%
  inner_join(get_sentiments("bing"), by = "word") %>%
  group_by(id) %>%
  summarize(sentiment_score = sum(ifelse(sentiment == "positive", 1, -1))) %>%
  ungroup()

captions_with_sentiment <- captions_for_analysis %>%
  left_join(sentiment, by = c("id" = "id"))

captions_with_sentiment
```

6) Visualization and interpretation
- Caption length distribution
- Top words across captions
- Sentiment by topic

```R
# Caption length distribution
ggplot(captions_with_sentiment, aes(x = word_count)) +
  geom_histogram(binwidth = 2, fill = "steelblue", color = "white") +
  labs(title = "Caption Word Count Distribution", x = "Words in caption", y = "Frequency")

# Top words across all captions (excluding stop words)
top_words <- captions_with_sentiment %>%
  unnest_tokens(word, caption) %>%
  anti_join(stop_words, by = "word") %>%
  count(word, sort = TRUE) %>%
  head(15)

ggplot(top_words, aes(x = reorder(word, n), y = n)) +
  geom_col(fill = "tomato") +
  coord_flip() +
  labs(title = "Top Words in Generated Captions", x = "Word", y = "Frequency")

# Sentiment by topic
sent_by_topic <- captions_with_sentiment %>%
  left_join(topics %>% select(id, topic), by = "id") %>%
  ggplot(aes(x = topic, y = sentiment_score, fill = topic)) +
  geom_col(show.legend = FALSE) +
  labs(title = "Sentiment Score by Topic", x = "Topic", y = "Sentiment score (positive - negative)") +
  theme_minimal()

print(sent_by_topic)
```

Best practices and safety notes
- Always set and protect your API key. Do not hard-code keys in notebooks or scripts.
- Be mindful of OpenAI usage costs: cap tokens and vary temperature to manage creativity vs. cost.
- Respect content safety: the meme prompts should avoid illicit, hateful, or harassing content. Use system prompts that steer away from unsafe topics.
- Document the version of the model and the exact prompts used to ensure reproducibility.
- If your institution has an OpenAI policy or a data-use policy, align the prompts and data handling accordingly.

What students should take away
- A practical, reproducible workflow to call the OpenAI API from R and integrate outputs with tidyverse workflows.
- Hands-on experience with prompt design, response parsing, and basic text analytics (tokenization, sentiment, word frequencies).
- Insight into how small changes in prompts can influence outputs and how to quantify those effects using tidyverse tools.
- A foundation for more advanced experiments in prompt engineering, model comparison, and content analysis in data science contexts.
