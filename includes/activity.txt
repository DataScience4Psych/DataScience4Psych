Meme-tastic OpenAI API & Tidyverse Lab for graduate data scientists

Learning goals
- Learn to call OpenAI’s API from R and handle chat completions in a tidyverse workflow.
- Build and manage prompts programmatically; iterate prompts and model parameters.
- Parse, clean, and store API responses in a tidy data frame; capture usage metrics (tokens, costs if you track them).
- Analyze and visualize results with tidyverse and tidytext (sentiment, length, model consistency).
- Reflect on model behavior, biases, and responsible usage in a data-science context.

- R (4.x) and an OpenAI API key (set as OPENAI_API_KEY in your environment).
- R packages: tidyverse, httr, jsonlite, purrr, stringr, glue, tidytext, tibble.
- Basic familiarity with dplyr/tidyr and ggplot2.

What you’ll build
- A reproducible, meme-inspired exploration: you generate short meme captions for a set of meme templates and topics, using OpenAI’s chat completions.
- A tidy workflow that stores prompts, responses, and metadata in a data frame.
- A small analysis report: caption length, sentiment, token usage, and cross-model or cross-prompt comparisons (where accessible).

4) OpenAI chat wrapper with retry/backoff
# Returns a list with: caption (string) and usage (list with tokens)
call_openai_chat <- function(prompt,
                             model = "gpt-3.5-turbo",
                             temperature = 0.7,
                             max_tokens = 60,
                             retries = 5,
                             backoff = 2) {
  url <- "https://api.openai.com/v1/chat/completions"
  body <- list(
    model = model,
    messages = list(
      list(role = "system", content = "You are a witty, meme-savvy data science caption assistant."),
      list(role = "user", content = prompt)
    ),
    temperature = temperature,
    max_tokens = max_tokens,
    n = 1
  )

  for (i in seq_len(retries)) {
    res <- tryCatch({
      POST(url,
           add_headers(Authorization = paste("Bearer", api_key)),
           content_type_json(),
           body = toJSON(body, auto_unbox = TRUE))
    }, error = function(e) NULL)

    if (is.null(res)) {
      Sys.sleep(backoff^(i-1))
      next
    }

    if (status_code(res) == 200) {
      content_text <- content(res, as = "text", encoding = "UTF-8")
      parsed <- fromJSON(content_text)
      caption <- parsed$choices[[1]]$message$content
      usage <- parsed$usage
      return(list(caption = caption, usage = usage))
    } else if (status_code(res) %in% c(429, 500, 502, 503, 504)) {
      Sys.sleep(backoff^(i-1))
      next
    } else {
      stop("OpenAI API error: ", status_code(res), " - ", content(res, as = "text", encoding = "UTF-8"))
    }
  }
  stop("Failed to fetch a response after retries.")
}

5) Generate captions for all memes
# Create prompts
memes <- memes %>% mutate(prompt = map2(template, topic, ~ build_prompt(.x, .y)))

# Call API for each meme (you can adjust model/temperature)
# Note: this will make multiple API calls. Ensure you have quota and API access.
model_choice <- "gpt-3.5-turbo"
temperature <- 0.75
results <- memes %>%
  mutate(api = map2(template, topic,
                    ~ call_openai_chat(build_prompt(.x, .y),
                                       model = model_choice,
                                       temperature = temperature))) %>%
  mutate(caption = map_chr(api, ~ .x$caption),
         usage = map(api, ~ .x$usage))

captions_df <- results %>%
  select(id, template, topic, prompt, caption, usage)

# Optional: extract total_tokens for analysis
captions_df <- captions_df %>%
  mutate(total_tokens = map_dbl(usage, ~ .x$total_tokens))

6) Quick sentiment (optional, using tidytext AFINN)
# Prepare for sentiment scoring (per-caption)
sentiment_scores <- captions_df %>%
  select(id, caption) %>%
  mutate(doc_id = row_number()) %>%
  unnest_tokens(word, caption) %>%
  inner_join(get_sentiments("afinn"), by = "word") %>%
  group_by(doc_id) %>%
  summarise(sentiment = sum(value, na.rm = TRUE))

# Join back to main data
captions_df <- captions_df %>%
  mutate(doc_id = row_number()) %>%
  left_join(sentiment_scores, by = c("doc_id" = "doc_id")) %>%
  select(-doc_id)

# If a caption has no sentiment words, sentiment will be NA; replace with 0
captions_df <- captions_df %>% mutate(sentiment = replace_na(sentiment, 0))

Tips and best practices
- Start with a small subset (3–4 memes) to iterate prompts quickly before scaling.
- Use a system message to set the persona and keep responses within the intended style and length.
- Always handle API usage data: total_tokens is valuable for budgeting and cost awareness.
- Respect content safety: include a discussion prompt in your class about model biases and ethical use; optionally implement a simple check against a blacklist of sensitive topics before posting captions.
