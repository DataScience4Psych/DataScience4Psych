Meme-tastic OpenAI + Tidyverse Lab for Grad Students

Overview
This graduate-level activity invites you to use OpenAI’s API from R, wrapped in a tidyverse workflow. You’ll generate meme-style captions for a set of topics, handle and parse API responses, and then analyze the results with tidyverse tools (tokenization, sentiment, embeddings, clustering, and visualization). The goal is to practice API calls, robust data handling, and rigorous analysis in a reproducible, tidyverse-friendly workflow.

Learning objectives
- Make OpenAI API calls from R (chat/completions and embeddings).
- Safely manage API keys and rate-limit handling in code.
- Build a reproducible, tidyverse-centric workflow to collect, clean, and analyze model outputs.
- Use text analysis (tokenization, word counts, sentiment) via tidyverse/tidytext.
- Use embeddings from the API to assess similarity and cluster generated captions.
- Create visualizations that compare prompts, models, and results.
- Reflect on model behavior, reliability, and ethical considerations of AI-generated text.

Prerequisites
- R (4.x) and RStudio (or any R IDE).
- Packages: tidyverse, httr, jsonlite, tidytext, textdata, bslib (optional for dashboards), ggplot2, tibble, dplyr, tidyr, purrr, stringr.
- An OpenAI API key. Set it as an environment variable OPENAI_API_KEY (or provide it securely in your session).
- A basic understanding of the tidyverse and dplyr verbs.

1) OpenAI helper functions (chat completions and embeddings)
- These functions handle the API calls, parsing, and basic error handling. They are designed to be robust to transient rate limits.

Code (R):
```r
library(httr)
library(jsonlite)

# Helper: safe POST with error handling
openai_post <- function(endpoint, body) {
  url <- paste0("https://api.openai.com/v1/", endpoint)
  res <- tryCatch({
    POST(url,
         add_headers(Authorization = paste("Bearer", Sys.getenv("OPENAI_API_KEY"))),
         content_type_json(),
         body = toJSON(body, auto_unbox = TRUE))
  }, error = function(e) NULL)

  if (is.null(res)) stop("OpenAI request failed (network/Error).")

  if (http_error(res)) {
    status <- status_code(res)
    content <- content(res, as = "parsed", type = "application/json")
    msg <- if (!is.null(content$error$message)) content$error$message else "Unknown error"
    stop(sprintf("OpenAI API error %s: %s", status, msg))
  }

  content(res, as = "text", encoding = "UTF-8") |> jsonlite::fromJSON(flatten = TRUE)
}

# Chat-completion helper (gpt-3.5-turbo or gpt-4-turbo)
openai_chat <- function(messages, model = "gpt-3.5-turbo", temperature = 0.7, max_tokens = 200) {
  body <- list(model = model,
               messages = messages,
               temperature = temperature,
               max_tokens = max_tokens)

  res <- openai_post("chat/completions", body)
  # extract content
  content <- res$choices[[1]]$message$content
  usage <- res$usage
  list(content = content, usage = usage, raw = res)
}

# Embeddings helper
openai_embeddings <- function(input_text, model = "text-embedding-ada-002") {
  body <- list(input = input_text, model = model)
  res <- openai_post("embeddings", body)
  embedding <- res$data[[1]]$embedding
  list(embedding = embedding, raw = res)
}
```

4) Prompt design and API call plan
- For each topic, generate multiple captions (e.g., 4 per topic) to explore variation across prompts and models.
- Use a consistent prompt skeleton with a target length and a request for multiple options.
- Example prompt: “Create 4 short, meme-style captions (no more than 12 words each) about the following topic. Use a witty, data-science-friendly tone. Label them Caption 1–Caption 4.”

Code (generate captions for each topic):
```r
library(purrr)
library(stringr)

# Prompt generator
make_prompts <- function(topic, n = 4) {
  rep(sprintf("Create %d short meme-style captions about: %s. Use a witty data-science tone and keep each caption under 12 words. Label as Caption 1, Caption 2, ...", 
              n, topic), 1)
}

# For each topic, call Chat API to generate 4 captions
generate_captions_for_topic <- function(topic_row, model = "gpt-3.5-turbo", n_caps = 4) {
  topic <- topic_row$topic
  prompts <- make_prompts(topic, n = n_caps)

  # We'll create a prompt that asks for 4 captions joined by "||" so we can split later
  user_prompt <- paste0("Please provide ", n_caps, " captions for the meme topic: ", topic,
                        ". Return exactly ", n_caps, " captions separated by the delimiter '|||'. Each caption should be a short, witty line under 12 words.")
  messages <- list(
    list(role = "system", content = "You are a witty data science meme assistant."),
    list(role = "user", content = user_prompt)
  )

  res <- openai_chat(messages, model = model, temperature = 0.6, max_tokens = 400)
  # split captions by delimiter
  caps <- str_split(res$content, "\\|\\|\\|", simplify = TRUE)[1, ]
  caps <- str_trim(caps)
  tibble(
    topic_id = topic_row$topic_id,
    topic = topic_row$topic,
    model = model,
    caption = caps
  )
}

# Run across topics (4 captions each)
captions_all <- pmap_dfr(list(memes, ~generate_captions_for_topic(.x, model = "gpt-3.5-turbo", n_caps = 4)),
                       .id = "rowid")

captions_all
```

Note: Depending on API behavior, you may need to adapt parsing. The important part is to collect a tidy data frame with topic_id, topic, model, caption.

6) Text analysis with tidyverse and tidytext
- Tokenize captions, compute basic statistics, and sentiment.

Code:
```r
library(tidytext)
library(textdata)

# Ensure we have a sentiment lexicon
# If needed, download lexicons:
# get_sentiments("bing"); get_sentiments("afinn")

captions_tokens <- captions_all %>%
  unnest_tokens(word, caption)

# Sentiment counts using Bing
sentiment_counts <- captions_tokens %>%
  inner_join(get_sentiments("bing"), by = "word") %>%
  count(topic, sentiment = sentiment, sort = TRUE)

# Summary by topic: word count, unique word count
summary_stats <- captions_all %>%
  mutate(word_count = str_count(caption, boundary("word"))) %>%
  group_by(topic) %>%
  summarize(mean_word_count = mean(word_count),
            max_word_count = max(word_count),
            n_captions = n()) %>%
  ungroup()

sentiment_counts
summary_stats
```

```r
embeddings_list <- map(captions_all$caption, ~openai_embeddings(.x, model = "text-embedding-ada-002")$embedding)

# Simple clustering (k-means)
set.seed(123)
k <- 3
km <- kmeans(embeddings_mat, centers = k)

captions_clustered <- captions_all %>%
  mutate(cluster = factor(km$cluster))
```


10) Memes, ethics, and reflection
- Meme angle: Encourage students to treat the captions as data, not as the final product. Discuss how meme culture shapes language, but model outputs can be biased or repetitive.
- Ethics quick-checks:
  - Ensure prompts avoid disallowed content and copyrighted character usage beyond fair use.
  - Be mindful of bias in generated content and in sentiment analyses.
  - Monitor API usage costs and rate limits; implement retry/backoff logic in production.
